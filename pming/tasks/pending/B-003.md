# Render deployment pipeline

Configure the Render Web Service for auto-deploy and create the health check endpoint.

## What to do

1. Create `src/app/api/health/route.ts` — checks DB connectivity, returns `{ status: 'ok', timestamp, version }`
2. Create or document `render.yaml` (Blueprint) with:
   - Web Service: build command, start command, health check path, env vars
   - PostgreSQL instance
   - Cron jobs (placeholders for now: stats update, magic link cleanup, weekly reminder)
3. Document the Render setup process for the team (which env vars to set, how to connect DB)
4. Ensure `pnpm build && pnpm start` works as Render expects

## Acceptance criteria
- GET /api/health returns 200 with status info
- App deploys successfully to Render on push to main
- Health check configured in Render dashboard
- Zero-downtime deploys working
- Env vars documented (which ones to set in Render)
