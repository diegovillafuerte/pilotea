# Remote parser config & kill switch

Ship parser specs over the air so a host-app UI change is fixed in hours, not Play-review days.

## What to do

1. Signed JSON config bundle (parser specs + min/max app versions + feature flags) served from a static endpoint (S3/R2/CDN — implementation choice)
2. `:sync` — fetch on app start + periodic WorkManager refresh; signature verification before applying; cached last-known-good fallback
3. Per-platform kill switch (disable a broken parser remotely; overlay shows "actualizando soporte para Uber" state)
4. Local spec override for development/testing
5. Version pinning: spec selection by host-app versionCode detected at runtime

## Acceptance criteria
- New spec applied without app update; bad signature rejected
- Kill switch silences a platform within one refresh cycle
- Offline drivers keep last-known-good specs
