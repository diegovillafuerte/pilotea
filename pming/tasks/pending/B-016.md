# Payment integration and subscription management

Integrate a payment processor for Pro ($59 MXN/mo) and Multi ($99 MXN/mo) subscriptions.

## What to do

1. Integrate payment processor (Stripe Mexico or Mercado Pago — decide at implementation time based on ease of setup for MXN recurring payments)
2. `src/app/api/subscriptions/route.ts` — POST: create subscription, GET: get current subscription
3. `src/app/api/webhooks/payments/route.ts` — webhook handler for payment events (subscription created, renewed, cancelled, failed)
4. `src/lib/db/queries/subscriptions.ts` — create, update status, check active
5. Update driver.tier when subscription status changes
6. Handle: subscribe, cancel, expire, renew flows
7. Subscription management UI (current plan, upgrade, cancel)

## Acceptance criteria
- Driver can subscribe to Pro or Multi tier
- Recurring monthly billing works
- Webhook correctly updates subscription status on payment events
- driver.tier reflects current active subscription
- Cancellation takes effect at end of billing period
- Expired subscriptions downgrade driver to free tier

## Dependencies
- E-001 complete (auth, dashboard working)

## Notes
- Decision between Stripe MX and Mercado Pago to be made at implementation time
- Stripe is simpler API but Mercado Pago has better adoption in Mexico
