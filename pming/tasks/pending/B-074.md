# UX polish: live simulator headline + tab re-tap pops to root

Two low-severity findings from the 2026-06-12 on-device session (`docs/didi-test-plan.md` F4/F5):

## What to do
1. **Simulator headline follows the live verdict.** `VerdictHeadline` picks its string by demo
   *shape* but tints by the live verdict level — after the driver moves the playground floor, the
   chip re-grades (e.g. amarillo) while the script still reads "Rojo: …". Pick the string by the
   live level (sim_verdict_good/marginal/bad keyed on `chipState.level`) so text and color agree.
2. **Tab re-tap pops the detail stack.** From a detail screen (Ayuda, Historial, Tu semáforo),
   tapping the bottom tab that is already selected does nothing. Make re-selecting the current tab
   pop to that tab's root (standard bottom-nav behavior).

## Acceptance criteria
- Playground changes never leave the headline contradicting the chip
- Tapping the active tab from any detail screen returns to the tab root

## Result

Both findings fixed (Android-only).

**F4 — simulator headline follows the live verdict.** Extracted `headlineResFor(level)` in
`SimulatorScreen.kt`: the guided headline now keys off `chipState.level`
(GREEN→verde / YELLOW→amarillo / RED→rojo) instead of the demo's fixed `shape`, so after the
playground slider re-grades the chip the text and colour always agree. The "¿por qué?" narrative
stays keyed to `shape` (it tells that scenario's pickup-distance story, which the slider doesn't
change). Robolectric test `VerdictHeadlineTest` asserts each level resolves to the matching es-MX
string.

**F5 — tab re-tap pops the detail stack.** `KomparaApp.kt` now resolves the highlighted tab from
the whole back stack via `KomparaDestination.activeTab(...)` (topmost tab route, falling back to
Inicio) so detail screens (Ayuda, Historial, Tu semáforo) keep their parent tab lit instead of
defaulting to Inicio. Re-tapping the active tab calls `popBackStack(route, inclusive = false)`,
popping the detail stack back to that tab's root (no-op at root). `ActiveTabTest` covers the
resolution logic.

Verification: `:overlay` + `:ui` `testDebugUnitTest` pass (incl. the 8 new tests); `:app:assembleDebug`
compiles. `pnpm lint` clean (0 errors). `pnpm test` shows only the pre-existing baseline failures
(unrelated legacy web/backend suites) — no new failures.

**Pending on-device check** (not possible in the worktree): confirm on a real device that (a) moving
the playground floor flips the headline word in lock-step with the chip colour, and (b) from Ayuda /
Historial / Tu semáforo, tapping the lit bottom tab returns to that tab's root.
