# Play Store submission package

Everything Play needs to approve an accessibility-powered driver tool — the make-or-break ops gate. Human task with assistant support.

## What to do

1. Play Console: app creation, store listing (Spanish-first; screenshots incl. overlay + simulator), content rating
2. **AccessibilityService API declaration form**: core-functionality justification (driver decision-support/distraction-reduction), demo video (record via B-037 simulator + live capture), `isAccessibilityTool=false` confirmed
3. Data-safety form: screen content processed on-device only; telemetry = anonymized counters; consistent with privacy policy (B-038)
4. Prominent-disclosure screenshots as review evidence
5. Internal → closed testing track setup; pre-launch report review
6. Contingency: documented appeal/resubmission plan if first review rejects (precedent: Mystro, Ruta Rentable, GigU all approved)

## Acceptance criteria
- Declaration submitted with video; app live on closed track
- Data-safety answers literally match app behavior
- Rejection playbook documented

## Dependencies
- B-036 (disclosure flow exists), B-037 (demo video source)
