# Database schema and Render Postgres setup

Define the Drizzle ORM schema for all 7 tables, generate the initial migration, and document Render Postgres setup.

## What to do

1. Define Drizzle schema in `src/lib/db/schema.ts` for all tables: drivers, magic_links, sessions, uploads, weekly_data, population_stats, subscriptions
2. Create the DB client/connection in `src/lib/db/index.ts` using postgres.js driver
3. Configure `drizzle.config.ts`
4. Generate and verify the initial SQL migration
5. Create the `get_percentile` SQL function (see tech design section 5.2) as a custom migration
6. Add all indexes defined in the tech design
7. Verify `pnpm db:generate` and `pnpm db:migrate` work against a local Postgres (or document the Render setup steps)

## Acceptance criteria
- Drizzle schema matches tech design section 5.1 exactly
- Initial migration creates all 7 tables with correct types, constraints, indexes
- get_percentile SQL function included in migration
- `pnpm db:migrate` applies cleanly to a fresh database
- DB connection works with DATABASE_URL env var
