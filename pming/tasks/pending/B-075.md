# In-app screen-reader start/restart + stopped notification

The OCR reader (MediaProjection) dies silently whenever Android revokes the projection — every
screen lock (`STOP_REASON_KEYGUARD`), app update, or process death — and the only way to restart it
is an adb command (`OcrConsentActivity`). A driver locks their phone between trips constantly, so
the chip silently stops appearing mid-shift (observed on-device 2026-06-12).

## What to do
1. Lector tab: "Lector de pantalla (DiDi)" section with live activo/detenido status and an
   Iniciar/Reiniciar button, shown once the accessibility reader is connected
2. Play-compliant prominent-disclosure dialog before triggering the system consent
   (TODO(legal-B038) copy review)
3. When the projection stops unexpectedly: post a "se detuvo el lector" notification that relaunches
   the consent flow on tap, and stop the foreground service so the "leyendo la pantalla"
   notification never lingers stale
4. Draft the B-065 Play Console texts (FGS mediaProjection justification, data-safety) in
   `docs/launch-runbook.md`

## Acceptance criteria
- Reader restartable entirely from the app (no adb)
- Locking the phone produces the stopped notification; tapping it re-prompts consent
- Lector tab reflects the real projection state
