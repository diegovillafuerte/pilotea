# Offer simulator

Show the magic before the first shift: replay fixture offer cards with the live overlay so the driver (and the Play reviewer) sees exactly what Kompara does.

## What to do

1. Simulator screen: full-screen mock Uber/DiDi offer cards rendered from fixtures (B-028 corpus)
2. Run the real parse → metrics → overlay pipeline against the mock cards
3. Guided script: 3 offers (good/marginal/bad) with explanation of each verdict
4. Threshold playground: driver adjusts $/km floor and re-evaluates the same offers
5. Used in onboarding (post-grant) and accessible from Ajustes; doubles as the demo for the Play declaration video

## Acceptance criteria
- Real pipeline (not hardcoded UI) produces the simulator verdicts
- Threshold changes re-grade offers live
- Recordable as the Play-review demo video

- B-028 (fixtures), B-031 (overlay)
