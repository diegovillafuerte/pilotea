# Fiscal page and PDF generation

Monthly earnings breakdown page and downloadable PDF report for the driver's accountant.

## What to do

1. `src/app/api/fiscal/route.ts` — GET: aggregate weekly_data by month and platform. Query: ?month=2026-03
2. `src/app/api/fiscal/pdf/route.ts` — GET: generate PDF using @react-pdf/renderer. Query: ?month=2026-03
3. `src/app/(app)/fiscal/page.tsx` — fiscal screen:
   - Month selector (dropdown, last 6 months)
   - Per platform: net earnings, gross earnings, total trips, commission, taxes
   - Monthly totals across platforms
   - "Descargar PDF" button
4. PDF layout: clean, printable format with:
   - Driver name, month, generation date
   - Table: platform breakdown (earnings, trips, commissions, taxes)
   - Totals row
   - Note: "Este documento es un resumen informativo, no un comprobante fiscal"

## Acceptance criteria
- Fiscal page shows correct monthly aggregation from weekly_data
- PDF generates and downloads correctly
- Handles partial months (driver didn't upload every week)
- Empty state for months with no data
- PDF is clean enough to hand to an accountant

## Dependencies
- B-010 (dashboard/weekly_data), at least 1 parser producing data
