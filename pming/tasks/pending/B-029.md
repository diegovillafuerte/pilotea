# Uber MX offer-card parser

The parser spec for Uber Driver (Mexico) offer cards: standard requests, Trip Radar, exclusive offers, surge, multi-stop, and reservation variants.

## What to do

1. On-device recon: capture node-tree fixtures from the current Uber Driver MX build for every card variant (use B-028 recorder); check whether Uber's native $/km badge appears and capture it too
2. Author the Uber parser spec: fare, pickup distance + ETA, trip distance + duration, rating, surge markers, card variant tagging
3. Handle the accept-countdown overlay states (card appears/updates/disappears)
4. Fixture corpus: ≥ 15 real (sanitized) cards covering all variants
5. Document Uber Driver MX package ID + version range in the spec

## Acceptance criteria
- ≥ 95% parse success across the fixture corpus
- Variant coverage: standard, Trip Radar, exclusive, surge, multi-stop, reservation
- No raw screen text leaves the device

## Dependencies
- B-027 (pipeline), B-028 (framework)
