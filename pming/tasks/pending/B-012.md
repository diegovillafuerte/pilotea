# DiDi screenshot parser

Parse DiDi earnings from 2 screenshots (earnings screen + tablero/dashboard). ~0.85 completeness.

## What to do

1. `src/lib/parsers/didi-screenshot.ts` — Claude Vision prompt for both DiDi screens
2. Send both images in a single Claude call, prompt cross-references data between them
3. Extract: earnings (net/gross), trips, $/km (DiDi reports natively), $/trip, $/hour, cash/card split, taxes, rewards
4. Derive: hours (estimated from earnings / $/hour)
5. Cannot extract: explicit commission (DiDi doesn't break it out) — null
6. data_completeness: ~0.85
7. Update parser router and upload UI to accept 2 files for DiDi

## Acceptance criteria
- Handles 2-image upload correctly
- Extracts all available DiDi fields including $/km (unique to DiDi)
- Estimates hours from $/hour when available
- Commission is null (DiDi doesn't report it)
- Tests with fixture screenshots
