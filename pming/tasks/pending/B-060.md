# On-device OCR engine (ML Kit)

Turn captured frames into recognized text blocks with screen bounds, on-device, using ML Kit Text Recognition.

## What to do
1. ML Kit text-recognition (bundled, on-device) wrapper: frame bitmap → list of `OcrTextBlock(text, RectBox bounds, confidence)`
2. Frame-to-bitmap from ImageReader (handle row stride/padding), downscale for speed
3. Throttle/debounce: one OCR pass per quiet window; skip identical frames (hash) to save battery
4. Map OCR output into a `ParserSnapshot`-compatible shape (text + bounds as nodes) so the existing spec engine / scrubber / recorder are reused
5. es-MX number/locale resilience in the recognized text

## Acceptance criteria
- A DiDi/inDrive offer screenshot yields the visible offer text with correct bounds
- OCR pass < ~300 ms on mid-range hardware; identical frames skipped
- Output flows into the same scrubber + spec engine as the node-tree path
