# Compare page

Side-by-side cross-platform comparison showing which platform pays better per metric.

## What to do

1. `src/app/api/compare/route.ts` — GET: return current week data for all platforms the driver uses, with winner per metric. For now, no paywall enforcement (that comes in E-003).
2. `src/app/(app)/compare/page.tsx` — comparison screen
3. `src/components/compare/compare-bar.tsx` — horizontal bar showing Platform A vs Platform B per metric, winner highlighted
4. `src/components/compare/platform-compare-card.tsx` — card grouping metrics for a platform pair

### Display rules
- Only compare metrics available on BOTH platforms being compared
- Show "No comparable" for metrics where one platform has null
- Winner = higher value (except commission where lower = better)
- If driver only uses 1 platform: show message "Agrega otra plataforma para comparar"

## Acceptance criteria
- Shows side-by-side comparison for multi-platform drivers
- Correctly identifies winner per metric
- Handles null metrics (only compares when both have data)
- Single-platform drivers see helpful empty state
- Mobile-friendly layout

## Dependencies
- B-010 (dashboard with weekly_data), at least 2 parsers implemented
