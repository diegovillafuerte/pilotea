# MediaProjection capture service & consent

Foreground service that captures the screen via MediaProjection for the SurfaceView platforms (DiDi/inDrive), with the per-session consent flow and the Android 14/15 capture-indicator/foreground-service-type requirements handled.

## What to do
1. `:ocr` module (new): MediaProjection capture service (`mediaProjection` FGS type), VirtualDisplay → ImageReader frames, throttled (~2–4 fps, only while a target app is foreground)
2. Per-session consent: launch `MediaProjectionManager.createScreenCaptureIntent()` from an Activity, persist the token for the session; re-prompt each session (Android 14+ requires it)
3. Only capture while DiDi/inDrive is the foreground app (pause otherwise — battery + privacy); stop on reader-off
4. Expose frames as a Flow for the OCR engine (B-060); never persist frames (debug recorder excepted)
5. Manifest: FGS type, POST_NOTIFICATIONS, capture notification channel

## Acceptance criteria
- Consent → capture → frames flow on device; indicator shown; stops when target app backgrounds
- No frame leaves the device; no persistence outside debug
- Survives screen rotation; clean teardown on reader-off
