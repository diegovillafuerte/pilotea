# Percentile engine port & seeds

Bring the web MVP's percentile system (city × platform × metric, linear interpolation, national fallback, commission inversion) to the new stack, seeded synthetic until real aggregates take over.

## What to do

1. Port `get_percentile` SQL + population_stats seeds (10 cities + national) from web repo into the greenfield backend (B-041 scaffolds it; this task completes + tests it)
2. Android `:metrics` percentile client: per-metric percentile for the driver's week, sample_size + is_national_fallback metadata, commission inversion for display
3. Percentile UI elements: PercentileBar (20-person icon visualization — port web component concept to Compose) + badges on metric cards
4. Free/premium gating hook (B-050 enforces; this exposes capability flags)

## Acceptance criteria
- Parity test: same inputs → same percentiles as web implementation
- City fallback to national when sample < 20
- Commission shows "lower is better" correctly everywhere
