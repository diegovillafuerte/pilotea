# Claude Vision import endpoint (port)

Port the web MVP's proven parsing pipeline (Uber weekly PDF ~0.95 completeness, Uber/DiDi/InDrive screenshots) to the greenfield backend as an import endpoint.

## What to do

1. Port parser prompts + Zod-equivalent validation from `src/lib/parsers/` (uber-pdf, uber-screenshot, didi-screenshot, indrive-screenshot) — prompts and fixtures are the asset; adapt to the new service
2. Import endpoint: file upload (10 MB cap, pdf/png/jpg/webp), parse via Claude Vision, return ParsedMetrics-shaped weekly data + completeness
3. Reuse image normalization learnings (sharp equivalent) and the week_start fallback fix from web history
4. Storage: R2/S3 bucket for originals (same key scheme), retention policy documented
5. Spanish error messages preserved

## Acceptance criteria
- Web-repo parser fixtures pass against the ported endpoint (parity test)
- Failure modes return the established Spanish errors
- Files private; processing server-side only for imports (distinct from on-device reader posture — documented)
