# AccessibilityService & event pipeline

The capture core: an AccessibilityService bound to the Uber and DiDi driver apps that turns window events into debounced node-tree snapshots for the parser layer. Read-only — no actions performed on host apps.

## What to do

1. `:capture` — `KomparaAccessibilityService` with config: `canRetrieveWindowContent`, event types `TYPE_WINDOW_CONTENT_CHANGED|TYPE_WINDOW_STATE_CHANGED`, `packageNames` filtered to Uber Driver + DiDi Conductor packages (verify current MX package IDs on-device)
2. Event debouncing/coalescing (50–100 ms) to tame content-changed storms; parse on background dispatcher
3. Node-tree snapshot utility: flatten `AccessibilityNodeInfo` tree to an ordered, immutable representation (text, viewId, role, bounds); recycle nodes correctly
4. Service health: detect disabled/killed service, expose state Flow for watchdog UI
5. `isAccessibilityTool="false"` in service declaration; strings for the Settings entry in Spanish
6. Instrumented test with a stub host activity proving event → snapshot flow

## Acceptance criteria
- Snapshots delivered < 50 ms after event on mid-range hardware
- Zero events received from non-target packages
- Service survives screen-off/Doze during an active shift (with whitelisting)
- No `performAction` calls anywhere in `:capture`
