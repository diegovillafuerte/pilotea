# Uber PDF parser

Parse the weekly earnings PDF from drivers.uber.com using Claude Vision. This is the richest data source (~20 fields, ~0.95 completeness).

## What to do

1. `src/lib/parsers/uber-pdf.ts` — implement parser:
   - Send PDF to Claude Sonnet API with vision
   - System prompt describes the Uber PDF layout: sections, field locations, expected data
   - Ask Claude to return JSON with all extractable fields
   - Validate response with Zod schema
   - Calculate derived metrics: earnings_per_trip, earnings_per_hour, trips_per_hour
   - Handle nulls: Uber never reports km, so earnings_per_km = null
   - Calculate data_completeness score
   - Return ParseResult (see types in tech design section 8)
2. `src/lib/parsers/types.ts` — ParseInput, ParseResult, ParsedMetrics types
3. `src/lib/claude/client.ts` — Claude API client wrapper for vision calls (reusable by all parsers)

## Fields to extract from Uber PDF
- Net earnings, gross earnings, total trips, hours online
- Platform commission (amount + percentage)
- Taxes, incentives, tips, surge earnings, wait time earnings
- Active days, peak day earnings, peak day name
- Week start date

## Acceptance criteria
- Given a real Uber PDF, extracts metrics correctly
- Derived metrics calculated accurately
- Zod validation catches malformed Claude responses
- data_completeness ~0.95 for a complete PDF
- Null for earnings_per_km (Uber doesn't report km)
- Clear error message if Claude can't parse the image
- Tests with fixture data (mocked Claude response)
