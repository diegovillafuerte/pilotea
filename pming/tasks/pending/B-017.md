# Paywall enforcement

Client-side and server-side paywall enforcement for Pro and Multi features.

## What to do

### Client-side
1. `src/components/layout/paywall-gate.tsx` — wrapper component that checks driver.tier and shows upgrade prompt if insufficient
2. `src/components/layout/upgrade-prompt.tsx` — paywall screen showing tier benefits and subscribe CTA
3. Wrap compare page with PaywallGate(multi)
4. Wrap personalized recommendations with PaywallGate(pro)
5. Upgrade prompts at natural moments: compare tab, recommendations section, tips page

### Server-side (the real gate)
6. Add tier check to `GET /api/compare` — return 403 if tier < multi
7. Filter recommendations in dashboard API based on tier (basic for free, full for pro+)
8. Middleware helper: `requireTier(tier)` for use in API routes

### Paywall rules (from business context)
- Cross-platform comparison: ALWAYS Multi-only (was never free)
- Personalized recommendations: Pro+
- Streak, basic percentiles, single-platform dashboard: always free
- Never remove a feature that was given free

## Acceptance criteria
- Free-tier driver sees upgrade prompt when tapping Compare tab
- Free-tier driver sees limited recommendations on dashboard
- API returns 403 for paywalled endpoints with wrong tier
- Upgrade prompts are contextual (show what they'd get)
- No feature previously free becomes paid
