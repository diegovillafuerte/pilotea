# Upload API and R2 storage integration

The upload API route that receives files, stores them in Cloudflare R2, creates upload records, and orchestrates the parsing pipeline.

## What to do

1. `src/lib/storage/r2.ts` — S3Client configured for R2, upload function (PutObject), download function (GetObject for re-processing)
2. `src/app/api/uploads/route.ts` — POST handler:
   - Auth check via middleware
   - Parse FormData (platform, upload_type, files[])
   - Validate: file size (<10MB), MIME type (png/jpg/webp/pdf), platform, upload_type
   - Upload to R2 with key `{driver_id}/{upload_id}.{ext}`
   - Create uploads record (status: processing)
   - Call parser (for now, only uber-pdf will exist — route to it)
   - On success: upsert weekly_data, update upload status to parsed
   - On failure: update upload status to failed with Spanish error message
   - Return metrics + percentiles + recommendations
3. `src/lib/db/queries/uploads.ts` — create, update status, list by driver
4. `src/lib/db/queries/weekly-data.ts` — upsert (by driver_id + platform + week_start)
5. `src/lib/parsers/index.ts` — router function: given platform + upload_type, call the right parser

## Acceptance criteria
- Files uploaded to R2 and retrievable
- Upload records tracked with status lifecycle (pending → processing → parsed/failed)
- weekly_data upsert is idempotent (re-uploading same week overwrites)
- File validation rejects oversized or wrong-type files with clear errors
- Spanish error messages for all failure modes
- driver.last_upload_at updated on successful parse
