# Dashboard page

The main screen: driver's current week metrics, percentile rankings, and basic recommendations.

## What to do

1. `src/app/api/dashboard/route.ts` — GET: return driver info, current week data per platform, percentiles, totals, recommendations
2. `src/app/(app)/dashboard/page.tsx` — dashboard screen
3. `src/app/(app)/layout.tsx` — authenticated layout with bottom navigation bar (5 tabs: Dashboard, Comparar, Subir, Fiscal, Tips). "Subir" tab visually prominent.
4. `src/components/dashboard/metric-card.tsx` — card showing metric name, value, percentile badge
5. `src/components/dashboard/percentile-bar.tsx` — 20 person icons visualization, driver's position highlighted
6. `src/components/dashboard/recommendation-card.tsx` — actionable tip card (positive/warning/info types)
7. `src/components/layout/bottom-nav.tsx` — 5-tab bottom navigation

### Dashboard content
- Header: total net earnings this week, streak badge if > 1 week
- 5 metric cards ($/viaje, $/km, $/hora, viajes/hora, comision %)
- Each card shows value + percentile + mini person-bar
- Null metrics show "No disponible" with explanation
- data_completeness indicator if < 100%
- 2-3 recommendation cards
- Empty state for new drivers with no uploads yet: "Sube tus datos para ver tu dashboard"

## Acceptance criteria
- Dashboard loads with real data after a successful upload
- Percentile visualization renders correctly on mobile
- Bottom nav works across all 5 tabs (other tabs can show placeholder content)
- Empty state for no-data drivers
- Responsive: works on 360px-414px width
- Spanish copy throughout

## Dependencies
- B-009 (percentile engine), B-007 (parser produces weekly_data)
