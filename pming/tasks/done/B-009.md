# Percentile engine and synthetic data seeding

Build the percentile calculation system and seed it with realistic synthetic data for the 10 priority cities.

## What to do

### Percentile engine
1. `src/lib/percentiles/engine.ts` — TypeScript wrapper that calls get_percentile SQL function for each non-null metric, returns PercentileResult[]
2. Handle commission % inversion (lower commission = higher percentile for display)
3. Handle null metrics (skip percentile calculation)
4. Return metadata: sample_size, is_national_fallback

### Seed data
5. `seed/population-stats.ts` — script to populate population_stats table with synthetic data:
   - 10 cities: cdmx, monterrey, guadalajara, puebla, toluca, tijuana, leon, queretaro, merida, cancun
   - Plus `national` fallback
   - For each city × platform × metric: p10, p25, p50, p75, p90, mean
   - Based on reasonable estimates (public reports, news articles, driver community data)
   - Sample sizes: 500-2000 for major cities, 100-300 for smaller ones
6. `pnpm db:seed` script runs the seeder

### Cron placeholder
7. `src/app/api/cron/update-stats/route.ts` — placeholder for future real-data aggregation (runs the SQL from tech design section 9). Protected by CRON_SECRET.

## Acceptance criteria
- Percentile engine returns correct percentile (1-99) for any metric value
- Falls back to national data when city sample < 20
- Synthetic data covers all 5 metrics × 3 platforms × 11 city groups
- Seed script is idempotent (can run multiple times)
- Commission percentile inverted for display (lower = better)

## Dependencies
- B-002 (database with population_stats table and get_percentile function)
