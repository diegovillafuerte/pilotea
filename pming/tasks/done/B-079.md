# Semáforo driven by the driver's preferred metric (IPK or IPH)

Product decision (2026-06-12, user-first): the traffic light currently blends both floors —
`NetProfitEngine.decideLevel` is GREEN only when **both** net $/km and net $/hr pass, YELLOW on any
mix — which reads as "off" to the driver. Drivers optimize ONE metric per shift: IPK (ingreso por
km, `netPerKm`) or IPH (ingreso por hora, `netPerHourEquivalent`). The semáforo color should be
decided by the driver's preferred metric alone. The preferred metric is the chip's main (top)
number; the other shows below as secondary.

## What to do
1. Settings: add `preferredMetric` (IPK | IPH, default IPK) + serialization; existing installs
   default to IPK with floors unchanged
2. "Tu semáforo" (ThresholdsScreen): toggle to pick the preferred metric, wired to the setting
3. `NetProfitEngine`: verdict level comes from the preferred metric's floors only. Keep computing
   and classifying the other metric (`netPerKmLevel`/`netPerHourLevel`) for display/explain. Keep
   guardrails: missing fare → RED; preferred metric untestable → fall back to the other metric,
   capped at YELLOW; missing inputs never GREEN
4. Overlay chip (`VerdictChipUi`): preferred metric big on top, the other secondary below; explain
   strings (KM_WEAK etc.) follow the preferred metric
5. Quick sheet (long-press): sliders edit the preferred metric's floors (today they are $/km-only)
6. Lector section (LectorScreen): quick-access shortcut into semáforo settings
7. Simulator: same engine, but verify the playground shows the same main/secondary ordering

## Acceptance criteria
- Toggling IPK ↔ IPH in "Tu semáforo" flips which metric colors the light and which renders on top,
  instantly and persisted across restarts
- Unit tests: same offer grades GREEN under one metric and RED under the other when floors say so;
  fallback (preferred metric untestable) caps at YELLOW
- Lector screen has a visible tappable shortcut that opens semáforo settings
- Existing installs migrate silently: default IPK, floors untouched

## Result
Shipped 2026-06-12 (commit 0f956bf). PreferredMetric (IPK | IPH, default IPK) in Settings with
silent migration; NetProfitEngine verdict decided by the preferred metric's floors alone (fallback
capped at YELLOW, fare/partial-data guardrails kept), Verdict stamped with the deciding metric;
chip leads with the chosen rate, quick sheet + simulator playground edit that metric's floors only;
"Tu semáforo" toggle; Lector shortcut; help copy updated. Codex full review (security/architecture/
correctness): one critical fixed — overlay snapshots now primed before the offer pipeline arms, so
the first offer after service start or an Ajustes edit grades with current values. History-grading
drift deferred as TD-025.
