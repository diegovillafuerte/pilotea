# WhatsApp magic link auth

Full authentication system: magic link generation, Twilio WhatsApp delivery, token verification, session management, and the login/verify UI.

## What to do

### Backend
1. `src/lib/auth/magic-link.ts` — generate cryptographic token, store in magic_links table with 15-min expiry, validate + mark used
2. `src/lib/whatsapp/client.ts` — Twilio WhatsApp client, send magic link message using pre-approved template
3. `src/lib/auth/session.ts` — create session (random token, store SHA-256 hash), verify session from cookie, destroy session
4. `src/lib/auth/middleware.ts` — reusable auth check for API routes (read cookie, verify session, attach driver_id)
5. `src/app/api/auth/login/route.ts` — POST: validate phone (E.164), rate limit (3 per 15 min), generate magic link, send via WhatsApp
6. `src/app/api/auth/verify/route.ts` — GET: validate token, find/create driver, create session, set cookie, redirect to /dashboard or /onboarding
7. `src/app/api/auth/logout/route.ts` — POST: delete session, clear cookie
8. `middleware.ts` (Next.js root) — protect /(app)/* routes, redirect unauthenticated to /login

### Frontend
9. `src/app/(auth)/login/page.tsx` — phone input with +52 default, "Enviar link por WhatsApp" button, "Revisa tu WhatsApp" waiting state
10. `src/app/(auth)/verify/page.tsx` — magic link landing, shows loading while verifying, handles errors

## Acceptance criteria
- Full login flow works: enter phone → receive WhatsApp → tap link → logged in
- Session persists for 30 days via HTTP-only Secure cookie
- Session tokens hashed in DB (SHA-256)
- Rate limiting prevents >3 attempts per phone per 15 minutes
- Unauthenticated users redirected to /login from any app page
- New drivers (onboarding_completed=false) redirected to /onboarding
- Logout clears session

## Dependencies
