# WhatsApp auth & anonymous-first accounts

Port the auth concept that fit drivers (no email — WhatsApp), but anonymous-first: the reader works with zero signup; an account unlocks sync, benchmarks, premium.

## What to do

1. Backend: WhatsApp OTP via Twilio (port web magic-link learnings; OTP fits native better than links), hashed session tokens, 30-day sessions
2. Android: anonymous device identity at install; "crear cuenta" flow (phone → OTP → done) only when the driver hits a sync/premium gate
3. Account merge: local anonymous data attaches to the new account
4. Profile basics: nombre, ciudad (10-city list + otras), plataformas
5. Logout/device management; session refresh

## Acceptance criteria
- Full reader + local stats usable with no account
- OTP flow < 60 s happy path; resend + fallback handled
- Anonymous → account merge loses no local data
