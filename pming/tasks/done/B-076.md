# Single semáforo shared by all platforms

Product decision (2026-06-12): drivers think in one acceptance floor for their shift, not one per
app — the per-platform thresholds (Uber/DiDi/inDrive selector in "Tu semáforo", per-platform
persistence, per-platform city seeds) are complexity without a user. Collapse to ONE global
threshold used by the overlay chip, the quick sheet, the simulator, and the editor.

## What to do
1. `Settings.thresholds: Map<Platform, PlatformThreshold>` → single `threshold: PlatformThreshold?`
2. Serialization: new platform-less keys; migrate legacy per-platform values (prefer DiDi — the
   OCR-live platform drivers actually tuned — then Uber, then inDrive)
3. `DefaultThresholds`: one row per city (the base/Uber multiplier row)
4. Drop the platform selector from ThresholdsScreen/ViewModel; overlay + simulator write the global
   threshold

## Acceptance criteria
- Editing the floor anywhere (editor, quick sheet, simulator playground) re-grades every platform
- Existing installs keep their tuned floors after update (legacy-key migration test)
- No platform parameter left in the thresholds API

## Result
Shipped 2026-06-12 (PR #6, commit 09eccf2). Single global threshold in Settings (new platform-less
keys; legacy values migrate DiDi → Uber → inDrive on read, covered by migration tests); city seeds
collapsed to the base row; platform selector removed from the editor; overlay quick sheet,
simulator playground, and editor share one floor.
