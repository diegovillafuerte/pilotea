# Parse telemetry & breakage alerting

Know when Uber/DiDi ship a UI change before drivers complain — without ever exfiltrating screen content.

## What to do

1. Anonymized counters only: parse attempts/successes/failures + card-variant tags, keyed by `(host app, versionCode, spec version)` — never raw text or node dumps
2. Local aggregation, batched upload via `:sync` (respect consent + offline)
3. "Reportar tarjeta no leída" button → consented, PII-scrubbed fixture submission (feeds B-028 corpus)
4. Backend side: simple ingestion + failure-rate alerting when a `(host, version)` pair spikes (hook to email/Telegram)
5. Privacy review against Play data-safety declarations

## Acceptance criteria
- Telemetry payloads contain zero screen-content strings (assert in tests)
- A simulated parser break on a new host version triggers an alert
- Driver fixture reports arrive sanitized and replayable

## Dependencies
- B-028 (parser events); backend pieces can stub until B-041 lands
