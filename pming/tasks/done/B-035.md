# inDrive offer parser (fast-follow)

inDrive's bid model makes per-offer $/km *especially* valuable (driver sets/accepts custom prices). Fast-follow after Uber + DiDi are stable.

## What to do

1. Recon fixtures from current inDrive MX driver build (bid list + counter-offer screens)
2. Parser spec: passenger offer amount, distances (pickup + trip), bid context; handle the multi-offer list layout
3. Verdict integration: evaluate each listed offer; overlay adapts to list-style cards
4. Fixture corpus ≥ 15 cards

## Acceptance criteria
- ≥ 95% parse success on fixtures; list cards each get a verdict
- No regression to Uber/DiDi paths (shared engine untouched or covered by tests)

## Dependencies
- B-029, B-030 (stable framework patterns to follow)
