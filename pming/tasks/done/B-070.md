# Verdict explainer + per-hour floors in Ajustes

The two P0 UX gaps from `docs/benchmark-ux-gaps.md`, executed 2026-06-12.

## What was done
1. **"¿Por qué?" explainer**: `NetProfitEngine` now exposes per-metric levels on `Verdict`
   (`netPerKmLevel`/`netPerHourLevel`); the expanded chip opens with one es-MX line naming which
   floor passed/failed ("Rinde por hora, pero deja poco por km.", "Paga bien el km, pero es
   lento…", etc.). No line when a metric was untestable — the missing-data hint covers that.
2. **Full threshold editor**: Ajustes → "Tu semáforo" (`ThresholdsScreen` + `ThresholdsViewModel` +
   pure `ThresholdEditor`) edits all four floors (verde/rojo × $/km/$/hr) per platform
   (Uber/DiDi/inDrive), persists immediately via the shared `SettingsRepository` (same store as the
   overlay quick sheet), and adds "Volver a la mediana de <ciudad>" — the first real consumer of
   the city-seeded `DefaultThresholds` table.

## Acceptance criteria
- ✅ Expanded chip explains the verdict in one es-MX line
- ✅ All four floors editable per platform from Ajustes; red ≤ green enforced; quick sheet in sync
- ✅ All unit tests green (engine levels, chip explain mapping, editor math)
