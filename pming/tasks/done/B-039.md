# Auto trip log & rollups

Turn the capture stream into the driver's automatic ledger: offers seen, trips taken, shifts, and day/week aggregates — the replacement for the web MVP's upload flow.

## What to do

1. Trip lifecycle inference in `:capture`/`:data`: offer card parsed → accepted (card dismissal + host state transition heuristics) → trip in progress → completed; record declined/expired offers too (acceptance-rate analytics)
2. Room persistence: offers, trips (with verdict shown vs outcome), shifts (auto-start/stop by activity windows)
3. Rollup jobs (WorkManager): daily/weekly aggregates per platform — earnings (gross/net), trips, km, hours, $/km, $/hr, acceptance rate; align week boundaries with web MVP convention (Monday start)
4. Goals & streaks: weekly target, consecutive-weeks streak (port web concept)
5. Reconciliation hooks for B-045 (imported weeks must merge idempotently)

## Acceptance criteria
- A simulated shift (fixture replay) yields correct trips/shift/day aggregates
- Declined offers tracked without polluting earnings
- Aggregates match the web MVP's weekly_data shape (field parity documented)

## Dependencies
- B-027 (capture), B-032 (metrics)
