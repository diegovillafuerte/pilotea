# Recommendations engine port

Port the web MVP's rule-based recommendations to captured data, with richer signals (acceptance rate, declined-offer economics, best hours).

## What to do

1. Port the 5 web rules (streak, high $/hr praise, low $/trip warning, high commission, cross-platform tip) into `:metrics`
2. New capture-powered rules: "rechazaste X ofertas que cumplían tu meta", "tus mejores horas son Vie 19–23", acceptance-rate guidance
3. RecommendationCard UI (positive/warning/info types — port component concept)
4. Placement: Inicio (2–3 cards) + Comparar verdict context
5. Unit tests per rule with fixture weeks

## Acceptance criteria
- Rules fire correctly on fixture data; no rule fires on insufficient data
- Spanish copy reviewed for driver tone (tú, direct, no jargon)
- Premium rules flagged for gating
