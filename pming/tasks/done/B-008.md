# Upload UI

The upload screen where drivers select their platform, pick a file, and see the parsing progress animation.

## What to do

1. `src/app/(app)/upload/page.tsx` — upload screen:
   - Platform tabs (only shows platforms driver selected during onboarding)
   - Per platform: brief instruction text + file picker
   - For Uber: show "PDF semanal (recomendado)" option
   - File picker with drag/drop on desktop, tap-to-select on mobile
   - Client-side validation (size, type) before upload
2. `src/components/upload/file-dropzone.tsx` — file selection component
3. `src/components/upload/processing-steps.tsx` — animated 4-step progress:
   - "Leyendo archivos..."
   - "Extrayendo datos con IA..."
   - "Calculando metricas..."
   - "Comparando con X conductores..."
4. Error state: show error message with retry button
5. Success state: redirect to dashboard with new data highlighted
6. `src/hooks/use-upload.ts` — upload state machine (idle → uploading → processing → success/error)

## Acceptance criteria
- Mobile-first design with large touch targets
- Platform tabs match driver's selected platforms
- File picker works on Android Chrome (primary target)
- Processing animation runs during the ~5-15s parse time
- Error messages shown in Spanish with retry option
- On success, navigates to dashboard

## Dependencies
- B-006 (upload API), B-005 (onboarding sets driver's platforms)
