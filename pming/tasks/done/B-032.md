# Net-profit metrics & verdict engine

The math that beats Uber's gross badge: per-offer net economics from the driver's cost profile, configurable thresholds, and the verdict decision.

## What to do

1. `:metrics` — pure-Kotlin engine (fully unit-testable, no Android deps):
   - Total distance = pickup leg + trip distance (drivers think this way; Uber's badge does too)
   - Gross $/km, $/min, $/trip; net variants after cost profile (fuel $/km from rendimiento + gas price, maintenance/insurance/rent prorated)
   - Verdict: green/yellow/red vs per-platform thresholds ($/km AND $/hr floors; configurable)
2. Threshold defaults per city seeded from web MVP percentile data (p50 as "yellow" floor)
3. Settings model in DataStore: thresholds per platform, cost profile inputs
4. Exhaustive unit tests incl. es-MX edge cases (missing fields → degraded verdict with what's available)

## Acceptance criteria
- Engine returns verdict + metric breakdown for any parsed card, even partial ones
- Cost profile changes reflect instantly in verdicts
- 100% unit coverage of verdict branching
