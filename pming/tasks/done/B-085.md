# Backend benchmarks: net_earnings + combined "all" population

The redesigned Comparar table (B-086/B-087) needs two benchmark reads that don't exist yet. Today `population_stats` carries only the 5 efficiency metrics, per single platform — so total earnings has no benchmark, and there's no way to rank a driver "vs all drivers" (the percentile engine ranks against one platform's population). Build these **backend-first** (the founder chose this over interim UI fallbacks).

See `docs/comparar-redesign-spec.md` §2 (data gaps) and §13 (Phase 2).

## What to do
1. **net_earnings metric.** Add `metric_name = "net_earnings"` to `population_stats` (weekly net per city/platform); seed p10/p25/p50/p75/p90 + mean per seeded city (synthetic to start, `is_synthetic = true`).
2. **Combined "all" population.** Add `platform = "all"` rows per `(city, metric)` — the population across all platforms — so a blended "Tú" value ranks against every driver, not just one app's. Seed for every metric incl. net_earnings.
3. **API.** Benchmarks endpoint returns the new rows; `GET /v1/benchmarks?city=&platform=all` works; national fallback (`city = "national"`) covers both.
4. **Android consumption.** `BenchmarksRepository` (and `DEFAULT_PLATFORMS`) fetch `"all"`; `PopulationStat`/`PopulationStatEntity` already carry mean + breakpoints (no schema change); add `net_earnings` to `MetricPercentiles` keys.

## Acceptance criteria
- Seeded cities have `net_earnings` rows AND `platform = "all"` rows for every metric.
- Android resolves: (a) Uber/DiDi `mean` per metric, (b) a percentile of a blended value vs the `"all"` population (national fallback when sample < 20).
- No regression to the existing per-platform percentile path (Inicio).

## Dependencies
- B-043 (benchmarks delivery/sync), B-046 (percentile engine + seeds). Ref `docs/comparar-redesign-spec.md`.

## Result (done 2026-06-16, branch `feat/comparar-redesign`)
- Seeds (`backend/seed/population-stats.ts`): added the `net_earnings` metric + a combined `all` platform. buildSeedRows now emits 264 rows (was 165): 66 `all` rows + 44 `net_earnings` rows; existing rows unchanged. Verified pure + `tsc --noEmit` clean.
- Fold (`src/jobs/fold-population-stats.ts`): folds `net_earnings` from real data and blends each driver's platforms into one `all` driver-week (sum totals, recompute rates) so the combined population accrues. Typechecks.
- Benchmarks route already serves any platform (incl `all`) — no change. Android `BenchmarksRepository.DEFAULT_PLATFORMS` now caches `all`; `:sync` compiles.
