# Wire OCR into the verdict pipeline

Unify the OCR capture source with the existing `OfferEvent` flow so the overlay, trip log, and telemetry work identically regardless of capture mode.

## What to do
1. Route OCR-derived snapshots through the same `OfferEventPipeline` (scrub → spec → OfferEvent)
2. Capture-mode selection per platform: Uber → accessibility nodes; DiDi/inDrive → OCR (driven by package + the §7 finding); avoid double-processing
3. Overlay/trip-log/telemetry collectors unchanged
4. Reader on/off controls both accessibility + MediaProjection consistently

## Acceptance criteria
- DiDi/inDrive offers produce the same overlay verdict path as Uber
- One offer = one OfferEvent regardless of source; no duplicates
- Reader toggle starts/stops both capture modes cleanly
