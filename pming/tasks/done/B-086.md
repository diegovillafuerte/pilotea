# Comparar data layer — ComparisonBuilder + ViewModel rewire

Build the comparison model and rewire `CompararViewModel` from a pair-comparison to a weekly benchmark hub. See `docs/comparar-redesign-spec.md` §3 and §4.

## What to do
1. **New types** (`:ui` `mx.kompara.ui.stats`): `ComparisonRow(metric, label, unit, tu, uberAvg, didiAvg, percentile, lowerIsBetter, uberNaReason, didiNaReason)` and `WeeklyComparison(weekStart, rows, standing, standingMetric, opportunities)`. Extend `CompareUiData` to carry `WeeklyComparison` for the 2+ and single-app modes.
2. **`ComparisonBuilder`** (new pure object, no Android): assembles the 6 rows from
   - **Tú (blended):** `PeriodStats.fromWeekly(weekRows, platform = null)`.
   - **Uber/DiDi averages:** `PopulationStat.mean` for `(city, platform, metric)`.
   - **Tu lugar:** `PercentileResult` (vs the `"all"` population from B-085); `topPercent`; commission inversion is automatic (`PercentileCalculator.INVERTED_METRICS`).
   - N/A reasons (Uber no km, DiDi no comisión, net no-benchmark resolved by B-085), standing pick order `earnings_per_hour → earnings_per_km → earnings_per_trip`.
3. **Rewire `CompararViewModel`:** inject `PercentileRepository`, `BenchmarksRepository`, `SettingsRepository` (keep `AggregateDao`, `TierGatekeeper`, `gateFunnel`). Mirror `InicioDashboardViewModel`'s flatMapLatest pattern (city from `settings.city`, percentiles via `PercentileRepository.observe(city, "all", MetricPercentiles.metricValues(blended))`). Drop `selectedPair`/`selectPair` and the pair-chooser path.
4. **Locked contract:** when `Capability.COMPARE` is locked, do NOT compose real premium values (the hero standing is computed free; table values withheld).

## Acceptance criteria
- `ComparisonBuilderTest`: blended Tú; N/A reasons; commission inversion (low cut → high displayPercentile); standing pick order; net-earnings handling.
- `CompararViewModelTest`: empty / single-app / full state derivation; city wiring; locked → no premium values composed; week resolution.

## Dependencies
- B-085 (the "all" population + net_earnings). Ref `docs/comparar-redesign-spec.md` §3–§4.

- `CompararViewModel` rewired: injects PercentileRepository + BenchmarksRepository + SettingsRepository; flatMapLatest over (week, city) → benchmarks + percentiles. Slimmed `CompareState` (availableWeeks + rowsForWeek). `MetricPercentiles.comparisonMetricValues` added (net + 4 rates).
