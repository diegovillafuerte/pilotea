# Region-based OCR parser specs

OCR has no view-ids and noisier structure than the node tree, so extend the parser spec/engine to match on text patterns + geometry (relative position, bounds clustering) over OCR blocks.

## What to do
1. Extend `ParserSpec`/`SpecEngine` (or an OCR variant) to extract fields from OCR blocks: anchored regex + geometry (above/below/left-of anchor, bounds clustering) — never absolute coordinates
2. Reuse the `OfferCard` output type and the fixture harness (OCR fixtures = recognized blocks + expected card)
3. Confidence handling: ignore low-confidence blocks; degrade gracefully on partial reads
4. Keep the node-tree path (Uber) unchanged

## Acceptance criteria
- OCR fixtures extract fare/distances/time into OfferCard at ≥95% on the corpus
- Geometry-based extraction robust to OCR ordering noise
- Node-tree specs/tests untouched and green
