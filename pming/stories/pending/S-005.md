# Additional Platform Parsers

Implement the remaining 3 parsers: Uber screenshot (limited pie chart view), DiDi (2 screenshots), and InDrive (1 screenshot). Each follows the same pattern as the Uber PDF parser but with platform-specific Claude prompts and validation.

## Tasks
- B-011: Uber screenshot parser
- B-012: DiDi screenshot parser
- B-013: InDrive screenshot parser

## Acceptance criteria
- Each parser extracts the correct fields per the data availability matrix in the tech design
- Null handling for metrics each platform can't provide
- data_completeness scores: Uber screenshot ~0.40, DiDi ~0.85, InDrive ~0.70
- Upload UI supports selecting upload type per platform
- DiDi flow accepts 2 images (earnings + tablero)
