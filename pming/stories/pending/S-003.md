# Upload & Parsing Pipeline

The core upload flow: driver selects a file, it's stored in R2, sent to Claude Vision for extraction, validated, and saved as structured weekly data. Phase 1 only implements the Uber PDF parser.

## Tasks
- B-006: Upload API and R2 storage integration
- B-007: Uber PDF parser
- B-008: Upload UI

## Acceptance criteria
- Driver can upload a PDF file from the upload screen
- File stored in Cloudflare R2 under driver's namespace
- Uber PDF parsed by Claude Vision → structured metrics extracted
- Derived metrics calculated ($/trip, $/hour, trips/hour)
- Data upserted into weekly_data (idempotent by driver + platform + week)
- Processing animation shows 4 steps while parsing runs (~5-15 seconds)
- Clear Spanish error messages on failure
- File validation: max 10MB, PDF/PNG/JPG/WebP only
